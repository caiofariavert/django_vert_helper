Objetivo desta biblioteca:
* Monitorá serviços como S3, Kafka e possíveis serviços que poderão ser definidos pelo settings
* Registrar ações ( funções ) que poderão ser vinculada como sugestão de execução no caso de falha de algum serviço, ou apenas serão ações para resolver/facilitar alguma questão no sistema
* Retornar via API (api/helper/v1/healthcare e  api/helper/v1/actions ) as ações e as informações de saúde no healthcare
* Teremos alguns funções de check de saúde para S3, Kafka e Postgres
* Usuário poderá setar qual função para realizar o check de saúde
* Disponibilizar saúde da aplicação em sí ( alteração no dockerfile provavelmente ) 
* Disponibilizar versão atual da aplicação ( commit )
* Saúde de aplicação (dockerfile) e commit deverão ser disponibilizados numa api em JSON se possível ( deverá ser acessível mesmo sem o Django estar funcionando ): api/helper/v1/app-health
* Deveremos ter uma documentação de como criar as ações, como configurar o check de saúde no settings, como deve retornar uma função personalizada de check de saude e como deve ficar o Dockerfile com as alterações( nossos dockerfile são padronizados )


Teremos ações programadas para aplicações e microsserviços que poderão resolver diversos gargalos
essas ações serão acionadas via API
Não penso em salvar isso em um banco de dados, mas se for facilitar o código, podemos armazenar
Caso use o banco de dados, crie uma PK com UUID
Caso não use, crie um campo SLUG para que possamos referenciar a função na API ao chama-las
